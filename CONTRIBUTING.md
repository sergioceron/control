# Contributing to Control

Thank you for considering contributing to the Control project. Below is a set of guidelines to help you make a meaningful contribution.

## Code of Conduct

This project and everyone participating in it is governed by the [Code of Conduct](CODE_OF_CONDUCT.md). By participating, you are expected to uphold this code.

## How Can I Contribute?

### Bug Reports

- Check the [issues](https://github.com/sergioceron/control/issues) to see if the bug has already been reported.
- If not, create a new issue using the bug report template.
- Provide detailed information and, if possible, a minimal reproduction of the issue.

### Feature Requests

- Use the [issue tracker](https://github.com/sergioceron/control/issues) to suggest new features.
- Provide clear problem descriptions and proposed solutions or approaches.

### Code Contributions

We welcome contributions in the form of pull requests (PRs).

#### Development Setup

1. Fork the repository and clone your fork locally.
2. Run `git remote add upstream https://github.com/sergioceron/control.git` to set up the upstream repository.
3. Create a new branch for your feature or bugfix.

#### Pull Request Process

1. Ensure the pull request (PR) has a meaningful title and description of your changes.
2. Follow the coding conventions outlined in our existing codebase.
3. Update documentation and add tests as necessary.
4. Ensure your PR passes all automated checks.
5. Submit your PR and reference any relevant issues in the description.

## Coding Standards

- Consistent use of code style as described in our past successful implementations.
- Include unit tests for new functionality
- Use clear and meaningful names for variables and functions.

## Testing

- Run all tests to ensure your contributions do not break existing functionality.
- Add tests for new features or bug fixes in respective directories.

## Submitting Issues

- Follow the issue templates provided in the issues section for consistency.
- Label your issues correctly to help maintainers manage and address them effectively.

---

We appreciate your contributions and thank you for helping to improve the project!
